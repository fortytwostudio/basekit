Links

Similar to the Basekit layout attribute, links in Basekit are also applied using a data attribute: data-bk-links.

This scoping to a data attribute allows simpler configuration and less stuff in the class attribute.

Links styles

Underline

This link has an underline.
Usage data-bk-links="primary underline"

Underline Hover

This link has an underline on hover.
Usage data-bk-links="primary underline-hover"

Underline Initial

This link initially has an underline and it disappears on hover.
Usage data-bk-links="primary underline-initial"

Dim

This link is dimmed until you hover.
Usage data-bk-links="primary dim"

Dark Background

This is a white link on dark background. Applying reliable color hover-states on dark background can be quite tricky unless the colour pallette has been designed specifically for this. For example, our primary and secondary styles don't look so good, particularly for WCAG compliant contrast ratios.
Usage data-bk-links="white underline-hover"

Dark Background Alternative

This is a white link on dark background dimmed. I find this is the best compromise of style and clarity.
Usage data-bk-links="white dim underline-hover"

Inherit

This link has inherited its parent text colour. When using inherit I find similar rules that apply to links on dark backgrounds also apply here.
Usage data-bk-links="inherit dim underline"
