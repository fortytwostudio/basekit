Reference

Basekit is a frontend framework designed to speed up the development of FortyTwo Studio web projects while balancing flexibility, optimisation, and consistency with the source graphic design.

You’ll need Gulp to optimise and build project files. Experience using Sass and Twig is required and you’ll need to configure the host and proxy values in the browserSync Gulp task if you want things to be awesome.

While templates are built with Twig for later integration with Craft CMS, the core framework doesn’t require Craft CMS.

Getting started

Assuming you’ve setup a local server (we rather ❤️ Laravel Valet) and you’ve downloaded Basekit to your project directory, cd into your project and run $ npm install to get the required packages.

Run $ gulp and the magic should start. At the moment this includes: Sass comiling to CSS, Twig compiling to HTML, Javascript to minified and combined basekit.js file, and finally browserSync will kick in. To make use of browserSync visit http://localdomain.dev:3000/demo/ and you should see the start page. Remember to configure browserSync the Gulp File.

If everything went to plan you should be up and running and ready to configure your Sass and Twig.

Directory Structure

  • css

    The core of Basekit, all your Sass is contained here.

  • demo

    Static templates and reference documentation get compile here.

  • imgs

    Logos, placeholders, UI images… but not content images unless the site will always be static.

  • js

    Numbered project javascript files compile to js/min/basekit.js.

  • node_modules

    Gulp dependencies, make sure this isn't committed to Git or you’ll not be happy.

  • templates

    Frontend template files, these compile to the demo/ directory.

  • bower_components

    Contains and keeps the html5shiv snippet up to date.
