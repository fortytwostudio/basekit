Reference

Basekit is a frontend framework designed to speed up the development of FortyTwo Studio web projects while balancing flexibility, optimisation, and consistency with the source graphic design.

You‚Äôll need Gulp to optimise and build project files. Experience using of Sass and Twig is expected and you‚Äôll need to configure the host and proxy values in the browserSync Gulp task if you want things to be awesome.

Getting started

While the frontend templates are built with, Twig to streamline the integration with Craft CMS templates, the core framework does not require Craft CMS to be used.

You‚Äôll also want a local server of some kind, we currently prefer Laravel Valet for small static sites and initial frontend templating, like this.

Once you've downloaded Basekit, cd in to the directory and run $ npm install to get the required Gulp packages.

Run $ gulp and it should start watching and compiling the relevant files, at the moment this includes: Sass (to .css), Twig (to .html), HTML (to minified and cleaned .html) and finally Javascript to minified and combined .js. It should also open the default local domain (basekit.dev:3000/demo/) with BrowserSync.

Now that you've comfirmed that Gulp is running, you're ready to start configuring Sass files (and Twig if you're using that).
