Basekit is a collection of front-end files and techniques used to build EQ Design web projects as of mid-late 2015.

Before continuing, you should have a sound understanding of CSS specificity, Sass syntax, and BEM-like class naming.

You would also benefit from an understanding of Utility Classes and the ITCSS methodology that I aim to follow.

Coding Style

Do not break these rules:

  • Use soft-tabs with a two space indent.
  • Put a space after : in Sass/CSS property declarations.
  • Put line breaks between Sass/CSS rulesets.
  • Always nest Sass selectors.
  • Put spaces before { in rule declarations.
  • Place closing braces } of declaration blocks on a new line.
  • Use BEM-like class naming for modules/components.
  • Use HEX or rgba colours.
  • Use // for Sass comment blocks (instead of /* */).
  • Do not use zero value units (margin: 0; not margin: 0px;).
  • Be explicit, avoid unnecessary shorthand declarations (write margin-bottom: 20px; instead of margin: 0 0 20px;).

Grid system

The basekit grid system or scaffolding uses the data-bkg attribute as opposed to more traditional classes. This allows style and layout structure to remain separate from each other.

All column widths can be defined at each breakpoint defined in the $mq-breakpoints variable, this also includes up to: each breakpoint.

Padding spacer

The padding (spacer) class is used to space out blocks of content across the x-axis. This class will over-ride the basekit grid padding.

pad--small
pad
pad--large
pad--bottom
pad--top

Additional options include:

  • pad--small--bottom
  • pad--large--bottom
  • pad--small--top
  • pad--large--top
  • Classes can be combined, e.g. pad--top  pad--large--bottom will result in a normal top padding space and a larger bottom padding.
